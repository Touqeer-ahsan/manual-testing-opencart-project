# OpenCart Manual Test Cases

Manual testing project for **OpenCart**, a popular open-source e-commerce platform. This repository contains a structured set of test cases designed to validate core functionality across the application, written as part of my QA/SDET portfolio.

## 📌 Project Overview

- **Application Under Test:** OpenCart (demo e-commerce site)
- **Testing Type:** Manual / Functional Testing
- **Total Test Cases:** 70
- **Modules Covered:** 16
- **Tool Used:** Excel (color-coded by priority)

## 🎯 Objective

The goal of this project was to apply manual testing fundamentals to a real-world e-commerce application — writing clear, traceable test cases that a QA team could actually execute, covering both happy-path and edge-case scenarios.

## 🧪 Test Case Format

Each test case in the Excel sheet includes:

| Field | Description |
|-------|-------------|
| Test Case ID | Unique identifier (e.g. TC_LOGIN_001) |
| Module | Feature area being tested |
| Test Scenario | What is being validated |
| Pre-conditions | State required before execution |
| Test Steps | Step-by-step actions |
| Expected Result | What should happen |
| Actual Result | Observed outcome |
| Status | Pass / Fail / Blocked |
| Priority | High / Medium / Low (color-coded) |
